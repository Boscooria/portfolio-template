
# PORTFOLIO TEMPLATE – BOSCO ORIA

## Descripción del proyecto

Este proyecto consiste en el desarrollo de una **plantilla base para portfolios de diseño gráfico**, concebida como un sistema reutilizable y escalable, y no como una web cerrada.

La idea es crear una estructura sólida que funcione como:

- Punto de partida para futuros proyectos de clientes  
- Demostración de capacidades técnicas y de diseño  
- Sistema personalizable adaptable a distintos contextos  
- Plantilla profesional preparada para bifurcarse y evolucionar  

---
He dividido cada parte del proyecto en un archivo HTML distinto, ya que esta estructura facilita la organización del código, mejora la legibilidad y permite un mantenimiento más claro y escalable del sitio web. Al separar las distintas secciones (home, proyectos, about, contact, etc.), se consigue una mejor gestión de contenidos y se sientan las bases para futuras ampliaciones o adaptaciones del portfolio sin necesidad de rehacer la estructura general.

Además, esta decisión responde a una lógica profesional de desarrollo web, similar a la utilizada en proyectos reales, y permite reutilizar componentes comunes como la navegación o el footer, manteniendo la coherencia visual y funcional en todo el sitio.

## Estructura del proyecto

### index.html

Página de entrada al portfolio.

- Funciona como “las puertas” al proyecto  
- Introduce que se trata de una plantilla de portfolio de Diseño Gráfico  
- Botón de acceso al portfolio (home.html)

---

### home.html

Página principal del portfolio.

**Estructura:**

- Navegación fija  
  - Work · About · Index (Bosco Oria) · Contact · Instagram  
  - Versión móvil con menú hamburguesa  

- Hero  
  - Imagen de fondo  
  - Texto principal (“Graphic Designer”)  
  - Animación de servicios  
  - Texto descriptivo inferior  
  - Botón de acceso a trabajos  

- Slider de trabajos  
  - Imagen del proyecto  
  - Nombre del proyecto  
  - Tipo de trabajo  

- Trabajos destacados  
  - Imagen del proyecto  
  - Nombre  
  - Fecha  
  - Tres puntos clave del trabajo  

- Footer  
  - Suscripción a newsletter (email)  
  - Sitemap (navegación)  
  - Redes sociales  
  - Información técnica (versión, hora local y estado)  
  - Nombre / marca  
  - Copyright & credits  

---

### dazed.html (Página de proyecto)

Plantilla para proyectos individuales.

**Estructura:**

- Navegación fija  
  - Misma estructura que home.html  
  - Versión móvil con menú hamburguesa  

- Contenido del proyecto  
  - Botón “Back Home”  
  - Tipo de proyecto (ej. Editorial)  
  - Título del proyecto  
  - Datos: cliente · rol · año · herramientas  
  - Texto explicativo del proyecto  
  - Imagen principal del proyecto  
  - Dos imágenes de proceso con texto explicativo  
  - Imagen final (mockup / proyecto aplicado a la realidad)  

- Navegación entre proyectos  
  - Enlace a “Next project”  
  - Enlace creativo a 404.html  

- Footer  
  - Misma estructura que el resto del sitio  

---

### about.html

Página de presentación personal.

**Estructura:**

- Navegación fija  
  - Misma estructura general  

- Contenido  
  - Imagen personal a la izquierda (ocupando toda la altura del texto)  
  - Columna derecha con:  
    - Lugar de residencia  
    - Nombre  
    - Rol (Graphic Designer)  
    - Introducción personal breve  
    - Qué hago actualmente  
    - Skills  
    - Botón “Let’s Talk” (contact.html)  

- Footer  
  - Misma estructura común  

---

### contact.html

Página de contacto.

**Estructura:**

- Navegación fija  
  - Misma estructura general  

- Columna izquierda  
  - CTA para iniciar colaboración  
  - Texto descriptivo breve  
  - Redes sociales (Instagram, LinkedIn)  

- Columna derecha  
  - Formulario de contacto:  
    - Nombre  
    - Email  
    - Tipo de servicio  
    - Propuesta  
    - Botón de envío  

- Elemento destacado  
  - Email de contacto en gran tamaño  

- Footer  
  - Misma estructura común  

---

### 404.html

Página de error personalizada.

- Mensaje 404  
- Concepto: el portfolio como “lienzo para el artista”  
- Mientras carga la página, el usuario puede “diseñar”  
- Botón volver a inicio (index.html)
- Botón borrar todo (borrar lo diseñado)

## Uso de Inteligencia Artificial

He utilizado IA (Gemini) como apoyo para estructurar y planificar el proyecto desde el inicio. En concreto, me ha servido para dividir el trabajo en fases, ordenar los apartados de la web y definir una arquitectura básica (secciones, jerarquía de contenidos y flujo de navegación), de manera que pudiera avanzar con una hoja de ruta clara y coherente.

Además, la IA me ayudó a generar una base inicial de contenido y organización sobre la que después he ido construyendo la web, así como una primera aproximación a textos que posteriormente he revisado y adaptado.

A partir de esa base, el desarrollo final se ha realizado integrando lo aprendido en clase y recursos de trabajos anteriores, aplicando buenas prácticas que ya había utilizado en proyectos previos. En resumen, la IA ha funcionado como una herramienta de apoyo en la planificación y el arranque, mientras que las decisiones finales y la implementación se han construido con criterio propio y con los conocimientos adquiridos durante el curso.
